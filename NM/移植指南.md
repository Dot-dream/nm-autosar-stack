# NM 标准化模块 — 移植指南

> 版本: V1.0 | 日期: 2026-05-27 | 语言: 中文

---

## 1. 概述

NM（Network Management）标准化模块是一套独立、可移植的汽车网络管理组件，支持三种 NM 协议，按通道配置即可切换：

| 模式 | 配置常量 | 协议 | 状态 |
|------|----------|------|:---:|
| OSEK 直接 NM | `NM_MODE_DIRECT` | OSEK NM 4.2.2 逻辑环 (Alive / Ring / LimpHome) | ✅ 完整 |
| OSEK 间接 NM | `NM_MODE_INDIRECT` | 应用消息监控（不发送 NM PDU） | ✅ 完整 |
| AUTOSAR NM | `NM_MODE_AUTOSAR` | AUTOSAR Nm 4.x 广播 (CBV / 协调器) | 🟡 骨架 |

**设计原则**：
- **零平台依赖**：核心逻辑不依赖任何 MCU、RTOS、编译器特性
- **配置外置**：所有参数通过 `const Nm_ConfigType*` 结构体传入
- **vtable 多态**：三种 NM 类型通过函数指针表分发，新增类型无需修改适配层
- **可裁剪**：未启用的功能通过 `#define` 在编译期完全剔除

**适用平台**: STM32F103 (ARMCC V5) / GD32F303 (GCC 10) / 其他 Cortex-M MCU

## 3. 快速开始

### 3.1 复制模块

将 `NM/` 整个目录复制到目标项目的源码树中，例如：

```
YourProject/
├── Middleware/
│   └── NM/            <-- 复制到这里
├── ...
```

### 3.2 实现平台适配（仅需 3 样东西）

NM 模块只要求目标项目提供 **3 个东西**：

#### (1) 临界区保护

在项目的头文件（如 `app_cfg.h`）中定义以下两个宏：

```c
/* FreeRTOS 示例 */
#define Nm_EnterCritical()    taskENTER_CRITICAL()
#define Nm_ExitCritical()     taskEXIT_CRITICAL()

/* 裸机示例（关全局中断） */
#define Nm_EnterCritical()    __disable_irq()
#define Nm_ExitCritical()     __enable_irq()
```

> **注意**：必须是宏定义（`#define`），不能是普通函数声明。NM 模块在编译期通过 `#ifndef Nm_EnterCritical` 检查。

#### (2) 毫秒级系统 Tick

```c
/* FreeRTOS 示例 (portTICK_PERIOD_MS 通常为 1) */
#include "FreeRTOS.h"
#include "task.h"

uint32 Nm_Timer_GetTick(void)
{
    return xTaskGetCount() * portTICK_PERIOD_MS;
}

/* 裸机示例 (SysTick 中断中维护 g_msCounter) */
volatile uint32 g_msCounter;

uint32 Nm_Timer_GetTick(void)
{
    return g_msCounter;
}
```

#### (3) CAN 驱动接口

```c
/* CAN 发送 */
CanNm_ReturnType CanNm_Transmit(NetworkHandleType channel,
                                 const uint8* pduData,
                                 uint8 pduLength)
{
    /* 调用你的 CAN 驱动发送 */
    CanIf_Transmit(TXPDU_ID_NM, pduData, pduLength);
    return CANNM_E_OK;
}

/* CAN 控制器使能 / 禁能（用于睡眠 / 唤醒） */
void CanNm_ControllerEnable(NetworkHandleType channel)
{
    Can_SetControllerMode(CAN_CTRL_NORMAL);
}

void CanNm_ControllerDisable(NetworkHandleType channel)
{
    Can_SetControllerMode(CAN_CTRL_SLEEP);
}
```

> 这三个函数在 `CanNm/CanNm.c` 中以 `NM_WEAK` 弱符号提供空实现。你的目标项目只需定义同名强符号函数即可覆盖。

### 3.3 编写配置

在项目中创建一个配置源文件（如 `Nm_Lcfg.c`）：

```c
#include "Nm.h"

/* ===== 单个通道配置 ===== */
static const Nm_ChannelConfigType NmChannel_CAN0 = {
    .channelHandle        = 0,                    /* 通道句柄 (0..N-1) */
    .nodeId               = 0x0A,                 /* 本节点 CAN ID */
    .busType              = NM_BUS_CAN,           /* 总线类型 */
    .nmMode               = NM_MODE_DIRECT,       /* ★ OSEK 直接 NM */
    /* .nmMode            = NM_MODE_INDIRECT,     *//* 或 OSEK 间接 NM */
    /* .nmMode            = NM_MODE_AUTOSAR,      *//* 或 AUTOSAR NM (骨架) */

    /* ---- 线缆协议 ---- */
    .wireConfig = {
        /* 方式 A: OpCode 模式 (6ER1 风格, 所有 NM 消息用同一个 CAN ID) */
        .wireFormat     = NM_WIRE_FORMAT_OPCODE,
        .singleCanId    = 0x500,
        .pduOpCodeAlive    = 0x01,
        .pduOpCodeRing     = 0x02,
        .pduOpCodeLimpHome = 0x04,

        /* 方式 B: CAN ID 模式 (6AQ0 风格, 消息类型由 CAN ID 范围区分) */
        /* .wireFormat  = NM_WIRE_FORMAT_CAN_ID, */
        /* .canIdBase   = 0x400, */
        /* .canIdMax    = 0x4FF, */
    },

    /* ---- 定时器参数 (单位: 毫秒) ---- */
    .timerTyp            = 1000,    /* 典型 Ring / NM PDU 周期 */
    .timerMax            = 2000,    /* 最大收包超时 */
    .timerError          = 1000,    /* LimpHome 发送周期 */
    .timerWaitBusSleep   = 5000,    /* 等待总线休眠 (TWbs) */
    .timerTx             = 100,     /* 发送重试间隔 */
    .timerNmMsgCycle     = 0,       /* NM 消息周期偏移 (AUTOSAR NM 用, OSEK 忽略) */

    /* ---- 重试限制 ---- */
    .rxCountLimit        = 4,
    .txCountLimit        = 8,

    /* ---- 可选功能 ---- */
    .busLoadReductionActive = 0,    /* 总线负载降低 (OSEK 专用) */
    .immediateTxConfEnabled = 0,    /* 立即 Tx 确认 */
    .busSyncEnabled       = 0,      /* 总线同步 / 协调器 (AUTOSAR NM 用) */

    /* ---- CFMOTO 扩展定时器 (仅 hasCfmotoExtensions=TRUE 时有效) ---- */
    .hasCfmotoExtensions    = 0,
    .cfmotoTSleepRequestMin = 0,
    .cfmotoTLimpSleepReqMin = 0,
    .cfmotoTLimpHomeDTC     = 0,

    /* ---- 回调函数指针 (NULL = 不注册) ---- */
    .stateChangeCallback = NULL,
    .pduRxCallback       = NULL,
};

/* ===== 全局配置 ===== */
const Nm_ConfigType NmConfig = {
    .numChannels          = 1,              /* 通道数量 */
    .channels             = &NmChannel_CAN0, /* 通道数组指针 */
    .mainFunctionPeriodMs = 5,              /* Nm_MainFunction() 调用周期 (ms) */
};
```

### 3.4 实现回调（`Nm_Cbk.h`）

应用层必须实现以下回调函数（声明在 `Nm_Cbk.h` 中）：

```c
/* ★ 必须实现的状态通知回调 ★ */
void Nm_NetworkStartIndication(NetworkHandleType ch) {
    /* NM 消息在 Bus-Sleep 中收到 → 系统唤醒 */
}
void Nm_NetworkMode(NetworkHandleType ch) {
    /* 进入 Network Mode → 允许应用层通信 */
}
void Nm_PrepareBusSleepMode(NetworkHandleType ch) {
    /* 进入 Prepare Bus-Sleep → 应用层应停止发送 */
}
void Nm_BusSleepMode(NetworkHandleType ch) {
    /* 进入 Bus-Sleep → 可关闭 CAN 控制器进入低功耗 */
}
void Nm_StateChangeNotification(NetworkHandleType ch,
                                 Nm_StateType prev, Nm_StateType cur) {
    /* 状态变更通知 (含前后状态) */
}

/* 可选回调 (受编译开关控制) */
void Nm_PduRxIndication(NetworkHandleType ch) { }
void Nm_RemoteSleepIndication(NetworkHandleType ch) { }
void Nm_RemoteSleepCancellation(NetworkHandleType ch) { }
void Nm_TxTimeoutException(NetworkHandleType ch) { }
void Nm_LimpHomeIndication(NetworkHandleType ch) { }
```

### 3.5 周期调用

```c
/* 在项目初始化中 */
int main(void)
{
    /* ... 硬件初始化 ... */

    Nm_Init(&NmConfig);      /* 初始化 NM 模块 */

    /* ... 创建任务 ... */
}

/* 在周期性任务中 (例如每 5ms 一次) */
void Task_Nm(void)
{
    while (1) {
        Nm_MainFunction();
        vTaskDelay(pdMS_TO_TICKS(5));
    }
}

/* CAN 接收中断 / 回调中 */
void CanIf_RxIndication(PduIdType pduId, const PduInfoType* info)
{
    if (IS_NM_PDU(pduId)) {
        Nm_RxIndication(channel, info->SduDataPtr, info->SduLength);
    }
}

/* CAN 发送完成中断 / 回调中 */
void CanIf_TxConfirmation(PduIdType pduId)
{
    if (IS_NM_PDU(pduId)) {
        Nm_TxConfirmation(channel);
    }
}

/* CAN Bus-Off 中断 */
void CanIf_ControllerBusOff(uint8 controllerId)
{
    Nm_ControllerBusOff(channel);
}
```

## 5. NM 类型选择与适用场景

### 5.1 OSEK 直接 NM (`NM_MODE_DIRECT`)

**适用场景**: CAN 总线上的逻辑环网络管理。每个节点通过 Alive / Ring / LimpHome 消息协调总线休眠。

**特点**:
- 主动发送 NM PDU（Alive → Ring 循环）
- 逻辑环：每个节点有后继节点，Ring 消息在环上传递
- LimpHome：通信异常时进入降级模式
- 总线负载可控（Bus Load Reduction）

**配置要点**:
- `nmMode = NM_MODE_DIRECT`
- `timerTyp / timerMax / timerError / timerWaitBusSleep` 必须按节点数合理设置
- `rxCountLimit / txCountLimit` 控制重试/错误检测阈值

### 5.2 OSEK 间接 NM (`NM_MODE_INDIRECT`)

**适用场景**: 不发送专用 NM PDU，通过监控应用层 CAN 消息来判定总线是否活跃。

**特点**:
- **不发送 NM PDU**：完全静默，仅监听
- 收到任何应用消息 → 认为总线活跃 → 进入 AWAKE → NORMAL
- 超时无消息 → WAITBUSSLEEP → BUSSLEEP

**配置要点**:
- `nmMode = NM_MODE_INDIRECT`
- `timerMax` 用于应用消息超时（无消息超时进入休眠）
- `timerWaitBusSleep` 用于等待休眠确认
- 不需要配置 Alive/Ring/LimpHome 相关参数

### 5.3 AUTOSAR NM (`NM_MODE_AUTOSAR`)

**适用场景**: AUTOSAR 标准网络管理，广播式协调，支持 Partial Networking (PN)。

**特点**:
- 周期性广播 NM PDU（Control Bit Vector）
- 协调器同步多节点休眠
- 无逻辑环，无 LimpHome
- 支持 Repeat Message Request / Active Wakeup

> ⚠️ **当前状态：骨架实现**。核心状态机框架和 vtable 桩已就位，CBV 处理、Repeat Message 计数、协调器同步逻辑标记为 `/* TODO: AUTOSAR NM */`。可用于集成编译，但完整功能需要后续开发填充。

## 8. 常见问题

### Q1: 编译报 `Nm_EnterCritical() must be defined`

Nm_OsIf.h 要求这两个宏必须在包含 NM 头文件之前定义。确认 `#define` 放在 `#include "Nm.h"` 之前。

### Q2: 编译报 `unknown type name 'uint8'`

嵌入式项目需确保 `Std_Types.h` 在 include path 中。PC 测试通过 `-DNM_HOST_TEST` 自动提供 typedef。

### Q3: ARMCC V5 编译报 `__attribute__((weak))` 语法错误

`Nm_OsIf.h` 已提供 `NM_WEAK` 宏兼容 ARMCC / GCC / IAR。如果你的编译器不支持，在包含 `Nm_OsIf.h` 之前定义：

```c
#define NM_WEAK  /* 不支持弱符号，提供强定义 */
```

### Q4: 如何在同一 ECU 上使用两种 NM 类型？

为不同 CAN 通道配置不同的 `nmMode`：

```c
static const Nm_ChannelConfigType NmChannel_CAN0 = {
    .nmMode = NM_MODE_DIRECT,   /* CAN0: OSEK 直接 NM */
    /* ... */
};
static const Nm_ChannelConfigType NmChannel_CAN1 = {
    .nmMode = NM_MODE_INDIRECT,  /* CAN1: OSEK 间接 NM */
    /* ... */
};
static const Nm_ChannelConfigType* channels[] = { &NmChannel_CAN0, &NmChannel_CAN1 };
const Nm_ConfigType NmConfig = { 2, channels, 5 };
```

### Q5: AUTOSAR NM 什么时候可以完整使用？

骨架文件 `CanNm_Autosar.c` 已包含完整的状态机框架和 CBV 结构定义。搜索 `TODO: AUTOSAR NM` 标记即可找到待实现的逻辑点。主要待办：
1. CBV 完整构造与解析（RepeatMsgReq / ActiveWakeup / PN / CoordinatorSleep）
2. Repeat Message 计数器递减
3. 协调器模式下的多节点睡眠同步
4. Partial Networking (PN) 过滤器

## 9. 与原始项目的对应关系

| 标准化模块 | 6AQ0 原始代码 | 6ER1 原始代码 |
|-----------|--------------|--------------|
| `Nm.c` / `Nm.h` | `CanStack\COM\NmIf\Nm.c` (78KB) | 无（6ER1 无 NmIf 层） |
| `CanNm_Osek_Direct.c` | `CanStack\COM\OsekNm\OsekNm.c` (103KB) | `Osek_NM\src\OsekDirectNm.c` (70KB) |
| `CanNm_Osek_Indirect.c` | 无（6AQ0 无 Indirect NM） | `Osek_NM\src\OsekIndirectNm.c` (43KB) |
| `CanNm_Autosar.c` | 无（6AQ0 预留 `NM_BUSNM_CANNM_ENABLED` 但未实现） | 无 |
| `Nm_Timer.c` | `OsekNm.c` 内嵌 decrement 定时器 | `OsekDirectNm.c` 内嵌 decrement 定时器 |
| `Nm_OsIf.h` | `SchM_Nm.h` + `SchM_OsekNm.h` + `MemMap` | FreeRTOS 协作调度 |

标准化模块相比原始代码的主要改进：
- 代码量从 ~324KB 降低至 ~120KB
- 消除 if/else 分支，vtable 单行分发
- Nm_Timer 独立中间件替代内嵌 decrement 定时器
- 三种 NM 类型统一接口，按配置切换
- PC 端可测试（原始代码绑定 MCU 硬件）
