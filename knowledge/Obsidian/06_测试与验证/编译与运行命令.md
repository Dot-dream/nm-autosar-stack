# 编译与运行命令

> 属于 [[../00_MOC_总索引|MOC 总索引]] > **06_测试与验证**

PC 端测试无需嵌入式硬件，仅需 GCC（支持 C11）即可编译运行。

---

## GCC 编译命令行

在 `NM/` 目录下执行：

```bash
gcc -std=c11 -DNM_HOST_TEST -I. -IOsIf \
    test/test_nm_state.c \
    Nm.c \
    CanNm/CanNm.c \
    CanNm/CanNm_Osek_Direct.c \
    CanNm/CanNm_Osek_Indirect.c \
    CanNm/CanNm_Autosar.c \
    Nm_Timer/Nm_Timer.c \
    -o test_nm_all
```

### 编译参数说明

| 参数 | 说明 |
|------|------|
| `-std=c11` | 使用 C11 标准（匿名 union / struct 支持） |
| `-DNM_HOST_TEST` | 启用 PC 端 Host Test Shim 层 |
| `-I.` | 将 `NM/` 目录加入 include path（查找 `Nm.h` 等） |
| `-IOsIf` | 将 `NM/OsIf/` 加入 include path（查找 `Nm_OsIf.h`） |
| 7 个 `.c` 文件 | 测试入口 + NM Core + CanNm 3 模式 + 定时器 |

### 涉及的源文件

| 文件 | 角色 |
|------|------|
| `test/test_nm_state.c` | 测试用例入口（7 个测试 + shim 实现） |
| `Nm.c` | NM Core（vtable 分发、状态协调、PDU 缓冲） |
| `CanNm/CanNm.c` | CanNm 适配层（vtable 实例化） |
| `CanNm/CanNm_Osek_Direct.c` | OSEK Direct NM 状态机 |
| `CanNm/CanNm_Osek_Indirect.c` | OSEK Indirect NM 状态机 |
| `CanNm/CanNm_Autosar.c` | AUTOSAR NM 骨架 |
| `Nm_Timer/Nm_Timer.c` | 软件定时器池 |

### 可选：添加警告编译

```bash
gcc -std=c11 -DNM_HOST_TEST -I. -IOsIf \
    -Wall -Wextra -Werror \
    test/test_nm_state.c \
    Nm.c \
    CanNm/CanNm.c \
    CanNm/CanNm_Osek_Direct.c \
    CanNm/CanNm_Osek_Indirect.c \
    CanNm/CanNm_Autosar.c \
    Nm_Timer/Nm_Timer.c \
    -o test_nm_all
```

`-Wall -Wextra -Werror` 确保零警告通过（符合 AGENTS.md 规范）。

---

## 运行

```bash
./test_nm_all
```

### 预期输出

```
=== NM Module Host Test Suite ===

Test 1: Basic state transitions ... PASSED
Test 2: LimpHome entry on TMax ... PASSED
Test 3: LimpHome recovery on message ... PASSED
Test 4: Wake-up from Bus-Sleep ... PASSED
Test 5: Communication disable ... PASSED
Test 6: User data set/get ... PASSED
Test 7: Indirect NM ... PASSED

=== All tests PASSED ===
```

### 失败情况

如果某个 `assert()` 失败，GCC 会打印行号并终止。例如：

```
Test 2: LimpHome entry on TMax ... Assertion failed: state == NM_STATE_LIMPHOME,
  file test/test_nm_state.c, line 242
Aborted (core dumped)
```

## 新增测试用例时的编译修改

如果要新增测试文件（如 `test/test_nm_timer.c`），只需在命令行中追加：

```bash
gcc -std=c11 -DNM_HOST_TEST -I. -IOsIf \
    test/test_nm_state.c \
    test/test_nm_timer.c \        # 新增
    Nm.c \
    CanNm/CanNm.c \
    CanNm/CanNm_Osek_Direct.c \
    CanNm/CanNm_Osek_Indirect.c \
    CanNm/CanNm_Autosar.c \
    Nm_Timer/Nm_Timer.c \
    -o test_nm_all
```

**规则**：
- 新增测试的 `.c` 文件放在 `NM/test/` 目录下
- 测试用例命名约定：`test_<模块>_<场景>.c`
- 所有测试 `.c` 文件在同一个 GCC 命令中编译
- 确保 `#include <assert.h>` + 使用 `Test_Reset() / Test_AdvanceTime()` 等共用 helper

**如果测试文件需要独立编译**（不含 `main()`），可编写单文件测试：

```bash
gcc -std=c11 -DNM_HOST_TEST -I. -IOsIf \
    test/test_nm_timer.c \
    Nm.c \
    Nm_Timer/Nm_Timer.c \
    -o test_nm_timer
```

仅链接必要依赖，减少编译时间。

## 一键脚本参考

可将编译+运行封装为脚本（`NM/test/run.sh`）：

```bash
#!/bin/bash
set -e
cd "$(dirname "$0")/.."
gcc -std=c11 -DNM_HOST_TEST -I. -IOsIf \
    -Wall -Wextra \
    test/test_nm_state.c \
    Nm.c \
    CanNm/CanNm.c \
    CanNm/CanNm_Osek_Direct.c \
    CanNm/CanNm_Osek_Indirect.c \
    CanNm/CanNm_Autosar.c \
    Nm_Timer/Nm_Timer.c \
    -o test_nm_all \
    && ./test_nm_all
```

或 Windows 下 `.bat`：

```bat
@echo off
cd /d "%~dp0\.."
gcc -std=c11 -DNM_HOST_TEST -I. -IOsIf ^
    -Wall -Wextra ^
    test\test_nm_state.c ^
    Nm.c ^
    CanNm\CanNm.c ^
    CanNm\CanNm_Osek_Direct.c ^
    CanNm\CanNm_Osek_Indirect.c ^
    CanNm\CanNm_Autosar.c ^
    Nm_Timer\Nm_Timer.c ^
    -o test_nm_all.exe
if %ERRORLEVEL% == 0 test_nm_all.exe
```
