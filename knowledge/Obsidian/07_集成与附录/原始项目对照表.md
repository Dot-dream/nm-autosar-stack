# 原始项目对照表

> 属于 [[../00_MOC_总索引|MOC 总索引]] > **07_集成与附录**

标准化 NM 模块派生于两个实际产品项目（6AQ0 和 6ER1），下表列出文件级别的对应关系和改进。

---

## 文件映射表

| 标准化模块文件 | 6AQ0 原始代码 | 6ER1 原始代码 | 改进说明 |
|---------------|--------------|--------------|----------|
| `Nm.c` / `Nm.h` | `CanStack\COM\NmIf\Nm.c` (78KB) | **无 NmIf 层**（6ER1 无统一接口层，各模块直接调用 OsekNm） | 抽象统一接口，消除重复 |
| `Nm_ConfigTypes.h` | 分散在宏定义中 (`NmIf_HwCfg.h` 等) | 分散在 `OsekNm_Cfg.h` | 集中管理，类型安全 |
| `Nm_Cbk.h` | `Nm_Cbk.h` (类似) | 无独立回调头文件 | 标准化 12 个回调 |
| `Nm_Internal.h` | 无（内嵌在 Nm.c 中） | 无 | 新增内部运行时类型 |
| `CanNm.h` | `OsekNm.h` | `OsekNm.h` | 统一 CanNm 接口 |
| `CanNm.c` | 无（直接调用 OsekNm.c） | 无 | 新增 vtable 适配层 (369 行) |
| `CanNm_Osek_Direct.c` | `CanStack\COM\OsekNm\OsekNm.c` (103KB) | `Osek_NM\src\OsekDirectNm.c` (70KB) | 提取纯状态机，剔除硬件绑定 |
| `CanNm_Osek_Indirect.c` | **无**（6AQ0 不支持 Indirect NM） | `Osek_NM\src\OsekIndirectNm.c` (43KB) | 独立实现，按配置启用 |
| `CanNm_Autosar.c` | **无**（6AQ0 预留 `NM_BUSNM_CANNM_ENABLED` 宏但未实现） | **无** | 全新骨架，预留 AUTOSAR NM |
| `Nm_Timer.c` / `Nm_Timer.h` | `OsekNm.c` 内嵌 decrement 定时器 | `OsekDirectNm.c` 内嵌 decrement 定时器 | 独立定时器中间件，池化复用 |
| `Nm_OsIf.h` | `SchM_Nm.h` + `SchM_OsekNm.h` + `MemMap` (3 文件) | FreeRTOS 协作调度 | 统一为单一抽象头文件 |
| `test/test_nm_state.c` | **无**（绑定 MCU 硬件，无 PC 测试） | **无** | 新增 PC 端 7 用例测试 |

| 维度 | 6AQ0 原始 | 6ER1 原始 | 标准化模块 |
|------|:--------:|:--------:|:--------:|
| NmIf / Core 层 | 78KB (`Nm.c`) | — | 26KB (`Nm.c` + `Nm.h`) |
| OSEK Direct NM | 103KB (`OsekNm.c`) | 70KB (`OsekDirectNm.c`) | 45KB (`CanNm_Osek_Direct.c`) |
| OSEK Indirect NM | — | 43KB (`OsekIndirectNm.c`) | 18KB (`CanNm_Osek_Indirect.c`) |
| CAN 适配层 | — | — | 14KB (`CanNm.c` + `CanNm.h`) |
| 定时器 | 内嵌 (约 15KB) | 内嵌 (约 10KB) | 8KB (`Nm_Timer.c` + `Nm_Timer.h`) |

标准化模块代码量约为原始代码的 **37%**（相对 6AQ0）和 **53%**（相对 6ER1）。

## 关键改进汇总

### 架构改进

| 改进点 | 原始状态 | 标准化后 |
|--------|----------|----------|
| **分发机制** | `if/else` 或 `switch` 分支判断 NM 类型 | vtable 多态单行分发 |
| **配置方式** | 6AQ0 全宏 / 6ER1 结构体混合 | 统一为 `const` 结构体 |
| **定时器** | 内嵌 decrement 循环，与状态机耦合 | 独立 `Nm_Timer` 池，ONESHOOT / PERIODIC |
| **可测试性** | 绑定 MCU 硬件，无法 PC 测试 | PC 端 shim 层，7 用例零硬件验证 |
| **代码复用** | 两个项目独立维护，无共享 | 单一代码库，按配置切换 |
| **可移植性** | FreeRTOS / 裸机混合，平台强耦合 | 3 个函数 + 2 个宏即可适配 |
