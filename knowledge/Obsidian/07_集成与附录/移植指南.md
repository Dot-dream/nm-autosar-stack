# 移植指南

> 属于 [[../00_MOC_总索引|MOC 总索引]] > **07_集成与附录**

NM 标准化模块是一套独立、可移植的汽车网络管理组件。移植到目标嵌入式平台时，**仅需适配 3 样东西**。

---

## 摘要：仅需 3 样东西的适配

| # | 适配项 | 形式 | 示例 |
|:--:|--------|------|------|
| 1 | **临界区宏** | `#define` 宏 | `#define Nm_EnterCritical() taskENTER_CRITICAL()` |
| 2 | **毫秒 Tick 函数** | 函数实现 | `uint32 Nm_Timer_GetTick(void) { return g_msCounter; }` |
| 3 | **CAN 驱动函数** | 3 个弱符号覆盖 | `CanNm_Transmit` / `CanNm_ControllerEnable` / `CanNm_ControllerDisable` |

以上 3 样东西之外，NM 核心代码**零修改**即可编译运行。

---

## 设计原则速览

- **零平台依赖**：核心逻辑不依赖任何 MCU、RTOS、编译器特性
- **配置外置**：所有参数通过 `const Nm_ConfigType*` 结构体传入
- **vtable 多态**：三种 NM 类型通过函数指针表分发，无需 `if/else` 分支
- **可裁剪**：15 个 `#define` 编译开关在编译期剔除未用功能

适用平台：**STM32F103** (ARMCC V5) / **GD32F303** (GCC 10) / 其他 Cortex-M MCU

---

## 嵌入式集成 5 步速查表

| 步骤 | 操作 | 验证标准 |
|:--:|------|----------|
| **1** | 将 `NM/` 目录完整复制到目标项目（如 `Middleware/NM/`） | 15 个文件无遗漏 |
| **2** | 在项目头文件中定义 `Nm_EnterCritical()` / `Nm_ExitCritical()` 宏 | 编译无 `#error "Nm_EnterCritical() must be defined"` |
| **3** | 实现 `uint32 Nm_Timer_GetTick(void)` 返回毫秒级系统 Tick | 链接通过，无 `undefined reference` |
| **4** | 实现 `CanNm_Transmit` / `CanNm_ControllerEnable` / `CanNm_ControllerDisable` 三个函数 | 编译链接通过（覆盖弱符号空实现） |
| **5** | 编写 `Nm_ConfigType NmConfig` 配置 + 实现 12 个回调函数 + 周期调用 `Nm_MainFunction()` | 硬复位后 NM 状态机正常运行 |

---

## 步骤 2 详解：临界区宏

必须在 `#include "Nm.h"` **之前**定义：

```c
/* FreeRTOS 示例 */
#define Nm_EnterCritical()    taskENTER_CRITICAL()
#define Nm_ExitCritical()     taskEXIT_CRITICAL()

/* 裸机示例（关全局中断） */
#define Nm_EnterCritical()    __disable_irq()
#define Nm_ExitCritical()     __enable_irq()

#include "Nm.h"  /* ← 必须在宏之后包含 */
```

> `Nm_OsIf.h` 编译期通过 `#ifndef Nm_EnterCritical` 检查，未定义会触发 `#error`。

---

## 步骤 3 详解：毫秒 Tick 函数

```c
/* FreeRTOS 示例 (portTICK_PERIOD_MS 通常为 1) */
#include "FreeRTOS.h"
#include "task.h"

uint32 Nm_Timer_GetTick(void)
{
    return xTaskGetCount() * portTICK_PERIOD_MS;
}

/* 裸机示例 (SysTick 中断中维护 g_msCounter) */
volatile uint32 g_msCounter;

void SysTick_Handler(void) { g_msCounter++; }

uint32 Nm_Timer_GetTick(void) { return g_msCounter; }
```

NM 模块内部所有定时器均基于此 Tick 计算时间差，**不依赖 RTOS 定时器服务**。

---

## 步骤 4 详解：CAN 驱动函数

这三个函数在 `CanNm/CanNm.c` 中以 `NM_WEAK` 弱符号提供空实现。你的项目只需定义同名**强符号**函数即可覆盖：

```c
CanNm_ReturnType CanNm_Transmit(NetworkHandleType channel,
                                 const uint8* pduData, uint8 pduLength)
{
    CanIf_Transmit(TXPDU_ID_NM, pduData, pduLength);
    return CANNM_E_OK;
}

void CanNm_ControllerEnable(NetworkHandleType channel)
{
    Can_SetControllerMode(CAN_CTRL_NORMAL);
}

void CanNm_ControllerDisable(NetworkHandleType channel)
{
    Can_SetControllerMode(CAN_CTRL_SLEEP);
}
```

## 步骤 5 详解：配置 + 回调 + 周期调用

### 编写配置

在项目中创建 `Nm_Lcfg.c`：

```c
#include "Nm.h"

static const Nm_ChannelConfigType NmChannel_CAN0 = {
    .channelHandle        = 0,
    .nodeId               = 0x0A,
    .busType              = NM_BUS_CAN,
    .nmMode               = NM_MODE_DIRECT,   /* ★ 选择 NM 类型 */
    .wireConfig = {
        .wireFormat         = NM_WIRE_FORMAT_OPCODE,
        .singleCanId        = 0x500,
        .pduOpCodeAlive     = 0x01,
        .pduOpCodeRing      = 0x02,
        .pduOpCodeLimpHome  = 0x04,
    },
    .timerTyp              = 1000,
    .timerMax              = 2000,
    .timerError            = 1000,
    .timerWaitBusSleep     = 5000,
    .timerTx               = 100,
    .rxCountLimit          = 4,
    .txCountLimit          = 8,
};

const Nm_ConfigType NmConfig = {
    .numChannels           = 1,
    .channels              = &NmChannel_CAN0,
    .mainFunctionPeriodMs  = 5,
};
```

两种线缆协议模式可选：

| 模式 | `wireFormat` | 适用场景 |
|------|-------------|----------|
| **OpCode 模式** | `NM_WIRE_FORMAT_OPCODE` | 6ER1 风格，所有 NM 消息用同一个 CAN ID，通过 PDU 字节区分 |
| **CAN ID 模式** | `NM_WIRE_FORMAT_CAN_ID` | 6AQ0 风格，通过 CAN ID 范围区分消息类型 |

### 实现回调

`Nm_Cbk.h` 中声明的 12 个回调必须全部实现（至少空函数体），否则链接报 `undefined reference`。核心回调：

```c
void Nm_NetworkStartIndication(NetworkHandleType ch) { /* 收到消息 → 唤醒 */ }
void Nm_NetworkMode(NetworkHandleType ch)             { /* 进入 Network Mode → 允许通信 */ }
void Nm_PrepareBusSleepMode(NetworkHandleType ch)     { /* 准备休眠 → 停止发送 */ }
void Nm_BusSleepMode(NetworkHandleType ch)            { /* 进入 Bus-Sleep → 关闭 CAN 控制器 */ }
void Nm_StateChangeNotification(NetworkHandleType ch,
                                 Nm_StateType prev, Nm_StateType cur) { /* 状态变更 */ }
```

### 周期调用

```c
int main(void)
{
    /* 硬件初始化 */
    Nm_Init(&NmConfig);

    while (1) {
        Nm_MainFunction();
        delay_ms(5);  /* = mainFunctionPeriodMs */
    }
}

/* CAN 接收中断 */
void CanIf_RxIndication(PduIdType pduId, const PduInfoType* info)
{
    if (IS_NM_PDU(pduId)) {
        Nm_RxIndication(channel, info->SduDataPtr, info->SduLength);
    }
}

/* CAN 发送完成中断 */
void CanIf_TxConfirmation(PduIdType pduId)
{
    if (IS_NM_PDU(pduId)) { Nm_TxConfirmation(channel); }
}

/* CAN Bus-Off 中断 */
void CanIf_ControllerBusOff(uint8 controllerId)
{
    Nm_ControllerBusOff(channel);
}
```

## 三种 NM 模式选择

| 模式 | 常量 | 适用场景 | 发送 NM PDU | 状态机 |
|------|------|----------|:----------:|:------:|
| OSEK 直接 NM | `NM_MODE_DIRECT` | CAN 逻辑环 | ✅ 是 | 10 状态 |
| OSEK 间接 NM | `NM_MODE_INDIRECT` | 仅监听应用消息 | ❌ 否 | 8 状态 |
| AUTOSAR NM | `NM_MODE_AUTOSAR` | AUTOSAR 广播式 | ✅ 是 | 🟡 骨架 |

同一 ECU 上可混合使用：

```c
static const Nm_ChannelConfigType NmChannel_CAN0 = { .nmMode = NM_MODE_DIRECT, ... };
static const Nm_ChannelConfigType NmChannel_CAN1 = { .nmMode = NM_MODE_INDIRECT, ... };
static const Nm_ChannelConfigType* channels[] = { &NmChannel_CAN0, &NmChannel_CAN1 };
const Nm_ConfigType NmConfig = { 2, channels, 5 };
```
